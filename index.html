01 — Contexto del proyecto

Un prompt maestro diseñado para dar la misma calidad en cualquier IA.

Zuma Sport necesitaba identificar qué productos, marcas y categorías están pegando en el mercado peruano antes de decidir qué representar, empujar o descontinuar.

El problema no era la falta de información, era la inconsistencia. Cada IA interpretaba "dame tendencias deportivas" de forma distinta. GPT daba listas genéricas. Gemini se iba por narrativa. Claude analizaba en profundidad. Imposible comparar o confiar.

Este prompt resuelve esa variabilidad. Obliga a cualquier LLM a seguir una estructura idéntica —4 deportes, 3 segmentos por deporte, 12 tendencias exactas— y a anclar cada una a productos específicos del catálogo real de Zuma descubierto dinámicamente.

02 — Qué resolvimos

De respuestas inconsistentes a outputs comparables.

— Antes

Categorías genéricas

"Lentes de natación antivaho", "balones de entrenamiento", "canilleras premium". Útil para un blog, inútil para decidir inventario.

+ Ahora

Productos específicos del catálogo

"Spalding TF-250", "Arena Cobra Ultra Swipe", "Opro Platinum". Cada tendencia ancla un SKU real para acción de compras.

— Antes

Marcas inventadas o desactualizadas

La IA recordaba marcas de hace 2 años o inventaba líneas. Sin forma de verificar qué distribuye Zuma ahora.

+ Ahora

Descubrimiento dinámico del catálogo

Fase 0 obligatoria: la IA consulta Ripley, Juntoz, Linio antes de analizar. Catálogo siempre actualizado sin tocar el prompt.

— Antes

Sin evidencia cuantitativa

"Esta tendencia está creciendo" sin datos. ¿Cuánto? ¿Dónde? ¿Comparado con qué? Nadie sabía.

+ Ahora

Google Trends con 4 consultas fijas

Interés temporal + comparación + geografía + breakout queries. URLs explícitas. Mismo método en todos los modelos.

— Antes

Recomendaciones sin semáforo

Toda tendencia parecía una oportunidad. ¿Cuál priorizar? ¿Cuál descartar? Imposible accionar.

+ Ahora

Veredicto 🟢 🟡 🔴 obligatorio

+50% crecimiento pega. +10-49% observar. Estancado no pega. Umbrales cuantitativos. Cero ambigüedad.

03 — Manual de uso

Cómo el prompt se ejecuta en cualquier modelo.

El prompt trabaja en 3 fases secuenciales. Cada fase produce un output antes de pasar a la siguiente. Esto garantiza que ningún LLM salte pasos o mezcle contextos.

0

Descubrir catálogo

La IA consulta las tiendas oficiales de Zuma antes de empezar. Extrae marcas, líneas y rangos de precio que Zuma realmente distribuye hoy.

→ Ripley Zuma Sport
→ Juntoz Zuma Sport
→ Linio Zuma Sport
→ CMR Zuma Sport

Output: tabla resumen del catálogo
1

Investigar Trends

Para cada tendencia la IA ejecuta 4 consultas a Google Trends con URLs explícitas y parámetros fijos. Datos cuantitativos reales, no intuición.

Q1 → Interés temporal 12m
Q2 → vs competidor directo
Q3 → Geografía por departamento
Q4 → Queries en aumento

Parámetro geo=PE · date=today%2012-m
2

Generar 12 tendencias

Estructura fija: 4 deportes × 3 segmentos NSE. Cada tendencia con veredicto semáforo, acción comercial, canal y urgencia.

1 Natación · 2 Fútbol
3 Running · 4 Básquet

T1=NSE A · T2=NSE B · T3=NSE C
Checkpoint obligatorio por deporte
04 — Cómo ejecutarlo

Activar búsqueda web es clave.

El prompt funciona en cualquier LLM moderno, pero requiere acceso web para descubrir el catálogo de Zuma y consultar Google Trends en tiempo real. Esta es la guía específica por plataforma.

★
Mejora crítica

Activa "Deep Research" cuando esté disponible

Gemini, ChatGPT Plus y Claude ofrecen modos de investigación profunda. Activarlo multiplica la precisión del análisis de Google Trends y la profundidad del descubrimiento del catálogo. Toma 5-10 minutos en lugar de 1-2, pero produce datos comparables con un informe de consultora.

Claude
1. Abre una conversación nueva en claude.ai.
2. Activa el botón de búsqueda web (ícono de globo en la barra inferior). Para análisis más profundo, activa también "Research" si está disponible en tu plan.
3. Pega el prompt completo. Claude ejecutará Fase 0 descubriendo el catálogo y luego procederá.
4. Espera 1-3 minutos con búsqueda simple, o 5-10 minutos con Research. Claude mostrará sus búsquedas en tiempo real.
Recomendado
ChatGPT / GPT-4o
1. Usa ChatGPT Plus con modelo GPT-4o o superior (la versión free tiene acceso web limitado).
2. Activa "Deep Research" si está en tu plan (botón en la barra de entrada). Alternativamente asegúrate de que Browse esté activo.
3. Pega el prompt. Con Deep Research toma 5-10 minutos pero el output es mucho más preciso.
4. Si GPT se detiene en T6 o T9 (sin Deep Research), escribe "continúa" y seguirá.
Recomendado
Gemini
1. Abre gemini.google.com.
2. Activa "Deep Research" (botón azul con ícono de lupa en la barra de entrada). Este paso es crítico — sin Deep Research el análisis de Trends pierde profundidad.
3. Pega el prompt. Gemini generará primero un plan de investigación que debes aprobar antes de ejecutar.
4. La ejecución toma 5-10 minutos pero produce el output más preciso. Las citas [1][2] al final son fuentes verificadas, no ruido.
Recomendado · Deep Research
Perplexity
1. Perplexity es una IA de búsqueda — acceso web está siempre activo.
2. Usa el modo Pro si lo tienes (mejor precisión con Trends).
3. Pega el prompt. Perplexity citará automáticamente las fuentes de cada consulta.
4. Es el modelo más preciso para datos cuantitativos actuales.
Opción fuerte
Copilot (Microsoft)
1. Usa copilot.microsoft.com o la app integrada en Edge.
2. Búsqueda web activa por defecto (Bing).
3. Funciona con el prompt compacto o balanceado (el completo puede exceder su contexto).
4. Si se corta, pide continuar con el checkpoint del último deporte.
Usa versión balanceada
Modelos sin web (Llama, Mistral, GPT-3.5)
1. Sin acceso web, el Plan B del prompt se activa automáticamente.
2. La IA preguntará por el catálogo de Zuma. Copia y pega las marcas que Zuma distribuye como respuesta.
3. Los datos de Trends serán declarados como "Estimado por conocimiento".
4. Útil para brainstorm, no para decisiones de inventario.
Solo para brainstorm
05 — El prompt

Copia, pega y ejecuta.

Tres versiones optimizadas según el límite de contexto de cada modelo. El completo para modelos premium, el balanceado para uso general, el compacto para APIs limitadas o automatizaciones.


    


  







  
06 — Preguntas frecuentes

  
Dudas habituales.


  

    

      

        ¿Por qué las respuestas de GPT y Gemini se ven distintas aunque uso el mismo prompt?
        +
      

      

        La estructura debe ser idéntica (mismo orden de deportes, mismos segmentos, mismo schema). Lo que varía es el estilo narrativo y los datos cuantitativos según la calidad de búsqueda del modelo. GPT tiende a ser más sintético, Gemini más detallado con citas. Si la estructura varía, verifica que pegaste el prompt completo y que activaste búsqueda web.
      

    


    

      

        ¿El prompt funciona en español y en inglés?
        +
      

      

        Está optimizado para español porque el mercado objetivo es Perú y los términos de búsqueda en Google Trends funcionan mejor en español para productos locales. Si necesitas una versión en inglés para análisis de tendencias globales, los deportes y segmentos se adaptan pero los distritos y NSE son específicos a Lima.
      

    


    

      

        ¿Qué pasa si Zuma Sport agrega o cambia marcas en su catálogo?
        +
      

      

        El prompt se actualiza solo. No hay marcas hardcodeadas. La Fase 0 consulta las tiendas oficiales de Zuma cada vez que ejecutas el prompt, así que si mañana Zuma toma representación de una marca nueva, la próxima ejecución la detectará automáticamente y la incluirá en las tendencias.
      

    


    

      

        ¿Cada cuánto debería ejecutar el prompt?
        +
      

      

        Mensualmente es el ritmo ideal. Google Trends actualiza datos cada 24-48 horas y las tendencias de retail deportivo tienen estacionalidad mensual. Ejecuta el primer lunes de cada mes y compara contra el mes anterior para detectar aceleraciones y desaceleraciones. Para campañas puntuales (Cyber Wow, Fiestas Patrias, Navidad) ejecuta una ronda específica 4-6 semanas antes.
      

    


    

      

        ¿Puedo modificar los 4 deportes analizados?
        +
      

      

        Sí, pero con criterio. Los 4 actuales (Natación, Fútbol, Running, Básquet) cubren las marcas ancla del catálogo Zuma: Arena/Speedo, Gilbert/Opro, potencial representación, y Spalding. Si cambias a deportes sin marca correspondiente en el catálogo, el prompt generará muchas "POTENCIAL REPRESENTACIÓN" y perderá utilidad comercial inmediata (aunque gana como radar de expansión).
      

    


    

      

        ¿Qué hago cuando GPT se corta antes de terminar las 12 tendencias?
        +
      

      

        Escribe "continúa desde [N/12] manteniendo el schema". El prompt tiene contadores [X/12] y checkpoints ✓ por deporte justamente para facilitar la retomada. GPT-4o raramente se corta, pero GPT-3.5 y Copilot sí tienen este límite. Para esos modelos usa la versión balanceada o compacta del prompt.
      

    


    

      

        ¿Los datos de Google Trends son precisos al 100%?
        +
      

      

        Son indicadores de interés relativo, no de ventas absolutas. Google Trends muestra búsquedas, no transacciones. Sirven para detectar dirección y estacionalidad con precisión alta. Para validar volumen real, cruza con datos internos de Zuma (sell-in a canal, sell-out de marketplaces) antes de asignar inventario significativo.
      

    

  







  
07 — Cómo publicar esta web

  
Tres opciones gratis para poner este sitio online.

  
Esta página es un archivo HTML autocontenido. Puedes subirlo a cualquier servicio de hosting estático en menos de 5 minutos. Todas las opciones abajo son gratuitas para sitios pequeños como este.


  

    

      
Vercel

      vercel.com
      
La opción más sencilla. Arrastra el archivo HTML a vercel.com, obtiene una URL instantánea tipo zuma-sport.vercel.app. Permite dominios propios después.

    

    

      
Netlify

      app.netlify.com/drop
      
Netlify Drop: solo arrastras el archivo HTML a su web y listo. URL inmediata tipo zuma-sport.netlify.app. Cero configuración.

    

    

      
GitHub Pages

      pages.github.com
      
Ideal si Zuma ya usa GitHub. Creas un repo público, subes el HTML como index.html, activas Pages y tu sitio queda en zuma.github.io/radar.
